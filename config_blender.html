Shape Grammar en Blender

Aprende a usar Shape Grammar desde Blender: metodología de diseño, configuración de salidas, formas primitivas, transformaciones y reglas.

Integración con Blender

Una mirada rápida a cómo tener Shape Grammar integrado y cómo se ve el addon en Blender.

Addon de Blender

Panel de Propiedades

Panel integrado en las propiedades de cada objeto para configurar directamente las gramáticas de forma.

Instalación Simple

Descarga el archivo .zip del addon, ve a Edit → Preferences → Add-ons → Install, selecciona el archivo y actívalo.

Integración Directa

Integración completa en el workflow de modelado sin necesidad de salir de Blender.

Configuración de Salidas

Control completo sobre formatos de exportación y parámetros de render en el addon de Blender.

Samples

Define el número de variaciones que se generarán automáticamente. Es el equivalente al parámetro "samples" en el archivo JSON.

Output

Controla los formatos de archivo que se exportarán:

  • Blendfile: Archivo .blend nativo de Blender
  • Values: Archivo JSON con los valores utilizados en la generación
  • STL: Archivo STL optimizado para impresión 3D
  • Render: Imagen renderizada del producto final
  • Mosaic: Genera un PDF con la imagen de todas las generaciones

Render Config

Configuración avanzada para el renderizado de imágenes:

  • Render Scene: Escenas predefinidas (basic, empty, neon, one_light, one_light-top)
  • Material: Materiales disponibles (clay, clay_red, wood_bamboo)
  • Quality: Calidad del render (1-100)
  • Resolution: Resolución en píxeles [ancho, alto]

Configuración de Shape Grammar

Control detallado sobre formas, transformaciones y reglas para personalizar el comportamiento generativo.

General Settings

Configuración básica que controla el comportamiento general de cada componente:

  • Number of Transformations: Número máximo de transformaciones que se pueden aplicar por parte
  • Max Number of Forms: Número máximo de formas primitivas que se pueden seleccionar
  • Optional: Define si la parte aparece aleatoriamente en las generaciones
  • Ignore: Excluye completamente la parte del proceso de generación

Forms

Selección de formas primitivas disponibles para cada componente. Cada forma puede activarse o desactivarse independientemente.

Transformations

Configuración de las transformaciones que modifican las formas primitivas. Cada transformación incluye parámetros específicos como ejes, factores y valores máximos.

Rules

Sistema de reglas que controla cuándo y cómo se aplican las transformaciones. Las reglas se encuentran debajo de la sección de Transformations en la interfaz.

Importante: Las reglas deben escribirse manualmente (no hay botones de selección). Utiliza la sintaxis específica como "twist only_with cube", "bevel always *", o "rotation except sphere".

Nota

Para conocer más sobre las 13 formas primitivas disponibles, las 14 transformaciones y el sistema de reglas (only_with, except, always), puedes consultar las secciones correspondientes en la página de Introducción donde encontrarás explicaciones detalladas y ejemplos prácticos.

Metodología de Diseño Shape Grammar

Proceso estructurado de 3 etapas para desarrollar gramáticas de forma efectivas.

1

Selección de Referencias

  • Productos relacionados de la misma marca/línea
  • Productos del usuario objetivo
  • Productos aspiracionales
  • Formas del contexto o gustos del usuario
2

Desarrollo

  • Identificar formas y transformaciones en referencias
  • Identificar elementos comunes
  • Recrear referencias usando las gramáticas definidas
3

Evaluación

  • Generar diseños consistentes con referencias
  • Verificar coherencia formal
  • Iterar y refinar configuraciones
